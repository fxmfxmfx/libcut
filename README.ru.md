# libcut

**Самохостящийся приватный просмотрщик TikTok. Без рекламы, без трекинга, без аккаунта.**

libcut — это идея [redlib](https://github.com/redlib-org/redlib) / [libreddit](https://github.com/libreddit/libreddit), но для TikTok. Вы запускаете приложение на своём сервере, подписываетесь на авторов по `@username` и получаете чистую ленту новых непросмотренных видео — с комментариями, избранным, скачиванием, фото-каруселями и полными профилями авторов. Весь сетевой доступ к TikTok идёт через SOCKS5-прокси и [`yt-dlp`](https://github.com/yt-dlp/yt-dlp); никакие сторонние скрипты и пиксели TikTok никогда не выполняются в вашем браузере.

> ⚠️ Проект предназначен для **личного** использования. Уважайте авторские права и правила TikTok.

## Возможности

### Просмотр
- **Подписки на авторов** по `@username` — профиль и последние видео подтягиваются через `yt-dlp` и сохраняются локально.
- **Лента непросмотренных видео** — показывает только то, что вы ещё не смотрели; автоматически проверяет новые видео при открытии приложения.
- **Профили авторов** — аватар, отображаемое имя, описание и полная статистика (подписчики, подписки, лайки/сердечки, число видео), а также сетка всех сохранённых видео.
- **Поиск по @username** — одновременно по вашей локальной библиотеке и напрямую в TikTok; переключатель «Всё / Авторы / Видео».

### Воспроизведение
- **Видеоплеер** с нативными контролами, авто-лупом и поддержкой HTTP `Range` (перемотка).
- **Регулировка громкости** — всегда доступна, проявляется при наведении.
- **Фото-карусели (слайды)** для фото-постов TikTok — листание стрелками, точками и клавишами ←/→.
- **Комментарии** с именами авторов, аватарами, числом лайков и временем — с кнопкой обновления.
- **iOS-совместимый muted-autoplay** — плеер запускается без звука и включается после первого тапа (обход политики автоплея Safari).

### Библиотека
- **Избранное** — сохраняйте видео в локальную базу на потом.
- **Скачивание видео** на компьютер одной кнопкой (стримится с поддержкой HTTP `Range`).
- **Отметка просмотренным** — по отдельности у каждого видео или «Отметить все просмотренными» в профиле автора.
- **Загрузка старых видео** — пагинация по бэклогу автора через `--playlist-start` у `yt-dlp`.

### Приватность
- **Без аккаунта TikTok** — подписывайтесь и смотрите анонимно.
- **Без трекинга и рекламы** — ничего из фронтенда TikTok никогда не выполняется в вашем браузере.
- **SOCKS5-прокси для всего доступа к TikTok** — каждый запрос идёт через ваш прокси на уровне `yt-dlp`.
- **Серверный прокси картинок** — все аватары и превью переотдаются через `/api/tiktok/proxy-image` (CDN TikTok блокирует hotlinking).
- **Временный кэш видео очищается каждые 10 минут** (настраивается через `TIKTOK_CACHE_TTL_MIN`) — метаданные в БД сохраняются, устаревшие `.mp4` удаляются.

### Кастомизация
- **6 встроенных тем**: Default (Rose), Gruvbox, Catppuccin Mocha, Nord, Dracula, Light.
- **Кастомный акцентный цвет** из палитры (или любое CSS-значение через кастомный CSS).
- **Инъекция кастомного CSS** — вставляйте произвольный CSS, который переопределяет все встроенные стили и темы.
- **Английский / Русский UI** — переключение на лету без пересборки.
- Все настройки внешнего вида хранятся в базе, переживают рестарт контейнера и применяются мгновенно.

### Самохостинг
- **Docker** — запуск одной командой `docker compose up -d --build`.
- **SQLite** через Prisma (не нужна внешняя БД).
- **`yt-dlp` + `curl_cffi`** для TLS-имперсонации браузера (обязательно — TikTok блокирует наивные клиенты).
- **Headless Chromium** (`puppeteer-extra` + stealth-плагин) для извлечения комментариев (принимает GDPR consent, кликает на иконку комментариев, перехватывает подписанный ответ comment API).
- **Демо-режим** — просмотр UI со встроенными примерами без обращения к TikTok.

## Быстрый старт (Docker)

Это основной способ запуска libcut.

### Что нужно
- **Docker** и **Docker Compose** v2 — проверьте через `docker --version` и `docker compose version`.
- Рабочий **SOCKS5-прокси** с выходным IP в регионе, где доступен TikTok.

Всё остальное (`python3`, `ffmpeg`, `yt-dlp`, `curl_cffi`, `node`, `bun`, Chromium) устанавливается автоматически внутри образа.

### Шаги

1. **Клонируйте репозиторий:**

   ```bash
   git clone <your-repo-url> libcut
   cd libcut
   ```

2. **Отредактируйте `.env`** в корне проекта — выключите демо-режим и укажите прокси:

   ```dotenv
   DEMO_MODE=false
   TIKTOK_PROXY=socks5://user:pass@1.2.3.4:1080
   ```

3. **Соберите и запустите контейнер:**

   ```bash
   docker compose up -d --build
   ```

4. **Откройте** http://localhost:3000 в браузере.

Проверить статус:

```bash
docker compose ps
docker compose logs -f libcut
```

Остановить:

```bash
docker compose down
```

> ⚠️ **Требования к сборке:** первая сборка запускает Turbopack (`next build`) **и** скачивает Chromium для сервиса комментариев. Закладывайте **~2 ГБ свободной RAM** и **~3 минуты** на сборку. Итоговый размер образа — **~1.5 ГБ** (большую часть занимает Chromium).

> ℹ️ **Дефолт DEMO_MODE:** в поставляемом `.env` стоит `DEMO_MODE=true`, чтобы UI можно было посмотреть в песочнице. На сервере вы **обязаны** переключить в `false` (и указать `TIKTOK_PROXY`), иначе реальных запросов к TikTok не будет.

Подробное руководство по Docker — в [DOCKER.md](DOCKER.md) и [DOCKER.en.md](DOCKER.en.md).

## Конфигурация

Вся конфигурация — через переменные окружения, задаваемые в `.env` (Compose подхватывает автоматически) или в командной строке.

| Переменная | По умолчанию | Описание |
|---|---|---|
| `DEMO_MODE` | `false` (Docker) / `true` (локально) | `true` — отдаются встроенные демо-данные, реальный TikTok не запрашивается. `false` — реальные запросы через `yt-dlp` + прокси. |
| `TIKTOK_PROXY` | _(пусто)_ | SOCKS5-прокси для `yt-dlp` и HTML-скрейпера. Формат: `socks5://user:pass@host:port`. Можно переопределить в UI. |
| `TIKTOK_CACHE_TTL_MIN` | `10` | Время жизни временного кэша видео в минутах. По истечении `.mp4` удаляется (метаданные в БД сохраняются). |
| `TIKTOK_SOCKET_TIMEOUT` | `30` | Таймаут одного сетевого запроса `yt-dlp` в секундах. |
| `TIKTOK_RETRIES` | `3` | Число ретраев `yt-dlp` при сетевых ошибках. |
| `DATABASE_URL` | `file:/app/db/custom.db` | Путь к файлу SQLite-базы (внутри контейнера). |
| `TIKTOK_CACHE_DIR` | `/app/cache/videos` | Папка временного кэша видео (внутри контейнера). |
| `NODE_ENV` | `production` | Режим Node.js. |
| `YTDLP_PATH` | `yt-dlp` | Путь к бинарнику `yt-dlp` (переопределите при нестандартной установке). |
| `TIKTOK_UA` | Chrome 124 User-Agent | User-Agent, который отправляют `yt-dlp` и HTML-скрейпер. |

> 💡 **Рантайм-настройки:** **прокси, язык, тема, акцентный цвет, кастомный CSS и поведение авто-отметки просмотренным** также хранятся в базе и редактируются в панели **Настройки** — применяются мгновенно без пересборки и переживают рестарт контейнера. Значение из БД имеет приоритет над env (кроме `DEMO_MODE`, который доступен только через env).

## Настройка прокси

TikTok агрессивно блокирует запросы с дата-центровых IP, поэтому **весь** сетевой доступ к TikTok идёт через SOCKS5-прокси. Прокси передаётся в `yt-dlp` флагом `--proxy=`, а в HTML-скрейпер — через `socks-proxy-agent`.

### Формат

```
socks5://user:pass@host:port       # с авторизацией
socks5://host:port                 # без авторизации
http://user:pass@host:8080         # yt-dlp умеет и HTTP-прокси
```

### Где настраивать

Прокси задаётся в двух местах (можно комбинировать):

1. **Переменная `TIKTOK_PROXY`** в `.env` — значение по умолчанию, когда в БД ничего нет.
2. **Настройки → Прокси** в приложении — URL + тумблер «Использовать прокси», хранится в БД и применяется мгновенно без пересборки. Если URL в БД пустой, берётся значение из env.

Прокси **включён по умолчанию** (`proxyEnabled=true`). Выключите тумблер в Настройках, если нужно отладить прямые запросы.

### Прокси на хост-машине

Если прокси крутится на той же машине, что и Docker, из контейнера к нему нужно обращаться по `host.docker.internal` (не по `127.0.0.1`):

```dotenv
TIKTOK_PROXY=socks5://host.docker.internal:1080
```

На старых Linux может понадобиться флаг `--add-host=host.docker.internal:host-gateway` (современные Docker добавляют автоматически — `docker-compose.yml` в репозитории уже это поддерживает).

### Трюк с SSH-туннелем

Если есть SSH-доступ к серверу в нужном регионе, поднимите SOCKS5-прокси одной командой:

```bash
ssh -D 1080 -N user@remote-host
# затем в .env:
# TIKTOK_PROXY=socks5://host.docker.internal:1080
```

### Прокси включён по умолчанию

Даже если вы задали `TIKTOK_PROXY`, его можно выключить в рантайме через тумблер в Настройках для тестирования — тогда `yt-dlp` пойдёт напрямую (и TikTok, скорее всего, заблокирует запрос).

## Руководство по использованию

### Подписки
1. Откройте вкладку **Подписки**.
2. Нажмите **Подписаться**.
3. Введите `@username` автора.
4. libcut загрузит профиль + последние видео через `yt-dlp` (через ваш прокси) и сохранит их локально.

### Лента
- Вкладка **Лента** показывает только непросмотренные видео от ваших подписок.
- При открытии приложения автоматически проверяются новые видео.
- Кнопка **Проверить новые** — принудительное обновление.

### Поиск
- Строка поиска в шапке ищет одновременно по вашей библиотеке и в TikTok.
- Переключатель **Всё / Авторы / Видео** фильтрует результаты.
- Поиск в основном по @username (точное совпадение) — полнотекстовый поиск TikTok ненадёжно скрейпится.

### Видеоплеер
Клик по карточке видео открывает плеер:
- **В избранное** — добавить/убрать из локального избранного.
- **Скачать** — сохранить `.mp4` на компьютер.
- **Комментарии** — просмотр + обновление (подпись запроса — на стороне headless-браузера).
- **Слайды** — для фото-постов: стрелки, точки, клавиши ←/→.
- **Громкость** — слайдер.
- **Отметить просмотренным / Отметить новым** — переключение состояния seen.
- **Оригинал** — открывает настоящий TikTok (с диалогом-предупреждением о трекинге/рекламе).

### Профиль автора
Клик по аватару/имени автора открывает профиль:
- Аватар, отображаемое имя, описание, подписчики, подписки, лайки, число видео.
- **Проверить новые** — загрузить свежие видео.
- **Загрузить старые видео** — пагинация по бэклогу.
- **Отметить все просмотренными** — снимает «новые» бейджи для этого автора.
- **Открыть в TikTok** — внешняя ссылка с диалогом-предупреждением.

### Настройки
Вкладка **Настройки**:
- **Язык** — английский или русский.
- **Тема** — 6 встроенных палитр.
- **Акцентный цвет** — пресет-свотчи.
- **Кастомный CSS** — произвольный CSS, инъектируется в страницу.
- **Прокси** — тумблер и URL.
- **Поведение** — авто-отметка просмотренным при открытии.
- **Кэш** — время жизни кэша видео (минуты).
- **Опасная зона** — очистить все данные (подписки, избранное, комментарии, настройки).

## Как это работает

libcut — это Next.js 16 одностраничное приложение с небольшим REST API. Интересное лежит в `src/lib/tiktok/` и `mini-services/tiktok-comments/`.

### `yt-dlp` + `curl_cffi`
[`yt-dlp`](https://github.com/yt-dlp/yt-dlp) вызывается как child-process (`src/lib/tiktok/ytdlp.ts`) для получения списков видео, stream-URL и метаданных. Вместе с ним ставится `curl_cffi` для **имперсонации TLS-отпечатка браузера** — без него TikTok блокирует экстрактор с `Unexpected response from webpage request`. Все запросы `yt-dlp` идут через разрешённый SOCKS5-прокси флагом `--proxy=`.

### HTML-скрейпер
`src/lib/tiktok/scraper.ts` тянет HTML страниц TikTok (тоже через прокси через `socks-proxy-agent`) и достаёт JSON-блоб `__UNIVERSAL_DATA_FOR_REHYDRATION__`. Это **надёжнее, чем yt-dlp**, для:
- `nickname` автора (отображаемое имя), `avatarLarger`, `signature` (био),
- `followerCount`, `followingCount`, `heartCount`, `videoCount`,
- `stats` видео (playCount, diggCount, commentCount, shareCount),
- `imagePost.images[]` для **детекции фото-карусели**.

### Прокси картинок
CDN TikTok блокирует hotlinking (возвращает 403 на `<img>` с чужих доменов). `src/lib/tiktok/images.ts` переписывает каждый удалённый URL картинки на `/api/tiktok/proxy-image?url=...`; сервер сам тянет картинку через прокси и переотдаёт. Без этого аватары и превью не грузились бы.

### Сервис комментариев
Comment API TikTok требует подписи `a_bogus`, которую нетривиально реализовать. Вместо этого мини-сервис `mini-services/tiktok-comments/` использует **headless Chromium** (`puppeteer-extra` + stealth-плагин):
1. Загружает страницу видео.
2. **Принимает GDPR consent-баннер TikTok** (EEA-регионы прокси показывают consent-стену, блокирующую комментарии, пока не принято — это был долгое время главный блокер).
3. **Кликает на иконку комментариев** — TikTok из настоящего браузера отправляет подписанный запрос к comment API.
4. **Перехватывает ответ** — парсит payload `/api/comment/list` в `{ authorName, authorAvatar, text, likeCount, postedAt }`.

Сервис слушает на порту `3040` (внутренний — наружу не проброшен) и вызывается из Next.js API-роута `/api/tiktok/videos/[id]/comments`. Латентность первой загрузки — **~20–40 секунд** (запуск Chromium + загрузка страницы + consent + клик + скролл); последующие быстрее.

### Временный кэш
`src/lib/tiktok/cache.ts` скачивает видео по требованию в `cache/videos/` (или `TIKTOK_CACHE_DIR`). Фоновый таймер запускается каждую **минуту** и удаляет файлы старше `TIKTOK_CACHE_TTL_MIN` (по умолчанию **10 минут**). Метаданные в БД сохраняются — файл просто скачивается заново при следующем просмотре.

### База данных
SQLite через Prisma (`prisma/schema.prisma`), модели `Author`, `Video`, `Comment`, `Favorite`, `SearchHistory`, `Setting`. Таблица `Setting` хранит редактируемые в рантайме значения (язык, тема, акцент, кастомный CSS, URL прокси, тумблер прокси, авто-отметка), чтобы они переживали рестарт и не требовали пересборки.

## Кастомный CSS

Откройте **Настройки → Кастомный CSS** и вставьте произвольный CSS. Он инъектируется в страницу как отдельный тег `<style id="libcut-custom-css">` внутри `<head>`, применяется **после** всех тем, акцентного цвета и базовых стилей — то есть переопределяет всё.

### Ключевые CSS-переменные

Все переменные определены в [`src/app/globals.css`](src/app/globals.css). Самые полезные для переопределения:

| Переменная | Что задаёт |
|---|---|
| `--primary`, `--primary-foreground` | Акцентный цвет (кнопки, активные элементы) + текст на нём |
| `--background`, `--foreground` | Фон страницы и основной текст |
| `--card`, `--card-foreground` | Фон карточек и текст |
| `--border`, `--input` | Цвет рамок и полей ввода |
| `--ring` | Кольцо фокуса |
| `--sidebar`, `--sidebar-foreground` | Фон и текст сайдбара |
| `--muted`, `--muted-foreground` | Приглушённые блоки и текст |
| `--accent`, `--accent-foreground` | Акцент ховера/выделения |
| `--secondary`, `--secondary-foreground` | Вторичный цвет и текст |
| `--destructive` | Цвет опасных действий |
| `--radius` | Базовый радиус скругления (от него производятся `--radius-sm/md/lg/xl`) |

Темы задают переменные на селекторе `[data-theme="..."]` (например, `[data-theme="gruvbox"]`); базовая палитра — на `:root` / `.dark`.

### Пример

```css
:root {
  --primary: #ff6b6b;
  --radius: 1rem;
}

/* Более круглые карточки видео */
.video-portrait {
  border-radius: 1.5rem;
}

/* Переопределение для конкретной темы */
[data-theme="gruvbox"] {
  --primary: #fabd2f;
}
```

### Советы
- Цвета можно задавать в любом CSS-формате (`#hex`, `rgb()`, `oklch()` — все встроенные темы используют `oklch()` или `#hex`).
- Акцентный цвет из **Настройки → Акцент** также переопределяет `--primary` (плюс `--ring`, `--sidebar-primary`, `--sidebar-ring`), но **кастомный CSS приоритетнее**, если задано в обоих местах.
- Кнопка **Сохранить** применяет CSS мгновенно — без перезагрузки. Значение хранится в БД.
- Чтобы сбросить — очистите поле и нажмите «Сохранить».

## Локальная разработка (без Docker)

Нужны [bun](https://bun.sh), Python 3 с `pip` и `ffmpeg` в `PATH`.

```bash
# 1. Установить JS-зависимости + создать SQLite-схему
bun install
bun run db:push

# 2. Установить yt-dlp + curl_cffi (для TLS-имперсонации TikTok)
pip install yt-dlp curl_cffi

# 3. Установить ffmpeg (macOS: brew install ffmpeg; Debian/Ubuntu: sudo apt install ffmpeg)

# 4. Запустить мини-сервис комментариев (опционально, нужен только для комментариев)
cd mini-services/tiktok-comments
bun install
bun run index.ts   # слушает на http://localhost:3040

# 5. Запустить приложение в демо-режиме (встроенные данные, прокси не нужен)
DEMO_MODE=true bun run dev
```

Откройте http://localhost:3000.

Для реального доступа к TikTok выставите `DEMO_MODE=false` и рабочий `TIKTOK_PROXY`:

```bash
DEMO_MODE=false TIKTOK_PROXY=socks5://127.0.0.1:1080 bun run dev
```

| Команда | Действие |

## Решение проблем

- **Видео не грузятся.**
  Проверьте, что `DEMO_MODE=false` в `.env` и что `TIKTOK_PROXY` указан и работает. Смотрите логи:
  ```bash
  docker compose logs libcut | grep -iE 'proxy|yt-dlp|error'
  ```

- **Аватарки/превью не грузятся.**
  Все картинки проксируются через `/api/tiktok/proxy-image`, потому что CDN TikTok блокирует hotlinking. Проверьте, что прокси включён и доступен. Если прокси на хосте — используйте `host.docker.internal` (не `127.0.0.1`).

- **Комментарии медленные или пустые.**
  Сервис комментариев использует **headless Chromium**: загружает страницу видео, принимает GDPR consent, кликает на иконку комментариев и перехватывает подписанный ответ API. Первая загрузка занимает **~20–40 секунд** (запуск браузера + страница + consent + клик + скролл). Последующие быстрее. Если комментарии стабильно пустые:
  - Убедитесь, что Chromium установлен (в Docker-образе ставится автоматически; для локалки — `cd mini-services/tiktok-comments && bun install`).
  - Убедитесь, что сервис комментариев запущен на порту 3040.
  - GDPR consent принимается автоматически — но если TikTok поменяет flow, комментарии могут тихо сломаться. Попробуйте другой регион прокси.

- **Сборка падает с кодом `137`.**
  Это OOM-kill — сборке нужно **~2 ГБ свободной RAM** (Turbopack + скачивание Chromium). Добавьте памяти серверу или соберите на более мощной машине и сделайте `docker save` / `docker load` образа.

- **`bun install --frozen-lockfile` падает при сборке.**
  Локальный `bun.lock` рассинхронизирован с `package.json`. На хосте выполните `bun install`, закоммитьте обновлённый `bun.lock`, затем пересоберите.

- **Прокси на хосте не доступен из контейнера.**
  Используйте `host.docker.internal` вместо `127.0.0.1`:
  ```dotenv
  TIKTOK_PROXY=socks5://host.docker.internal:1080
  ```

- **Нет звука на iOS.**
  Safari блокирует автоплей со звуком. Тапните по видео (или по оверлею Play) — звук включится. Это сделано намеренно (muted autoplay + unmute on user gesture).

- **Фото-посты показывают ошибку.**
  Фото-карусели детектятся HTML-скрейпером из `imagePost.images[]` в JSON `__UNIVERSAL_DATA_FOR_REHYDRATION__`. Если скрейпер не достучался до TikTok (проблема с прокси, региональный блок, смена вёрстки TikTok), пост может провалиться в видеоплеер и зафейлиться. Проверьте прокси и пересоберите образ, чтобы получить свежий `yt-dlp`.

- **Хочу всё сбросить.**
  ```bash
  docker compose down -v && docker compose up -d --build
  ```
  Или через UI: **Настройки → Опасная зона → Очистить все данные**.

## Где хранятся данные

Docker Compose создаёт два именованных тома (см. [`docker-compose.yml`](docker-compose.yml)):

| Том | В контейнере | Что хранит |
|---|---|---|
| `libcut-db` | `/app/db` | SQLite-база `custom.db` — подписки, видео, избранное, комментарии, настройки |
| `libcut-cache` | `/app/cache/videos` | Временные `.mp4` (чистятся по TTL, по умолчанию 10 минут) |

Посмотреть пути на диске:

```bash
docker volume inspect libcut_libcut-db
docker volume inspect libcut_libcut-cache
```

Стереть всё (вы потеряете подписки + избранное + комментарии + настройки):

```bash
docker compose down -v
```

## Обновление yt-dlp

TikTok часто меняет внутренний API, поэтому `yt-dlp` нужно периодически обновлять. Простейший путь — пересобрать образ: Dockerfile тянет свежий `yt-dlp` через `pip3 install` на каждой сборке:

```bash
docker compose up -d --build
```

Чтобы закрепить конкретную версию для стабильности, отредактируйте [`Dockerfile`](Dockerfile):

```dockerfile
RUN pip3 install --no-cache-dir --break-system-packages yt-dlp==<version> curl_cffi
```

Если экстракция внезапно сломалась на рабочей установке — первое, что стоит попробовать, это пересобрать образ: мейнтейнеры `yt-dlp` обычно выкатывают фикс в течение часов после поломки со стороны TikTok.
