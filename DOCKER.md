# Развертывание в Docker

> Русская версия | [English version](DOCKER.en.md)

Самохостящийся парсер TikTok без трекеров (аналог redlib для TikTok).
Приложение работает на Next.js 16 + Prisma (SQLite), получает видео через
`yt-dlp` + `ffmpeg`, и **весь** сетевой доступ к TikTok идёт через SOCKS5-прокси.

---

## Что нужно

- Сервер с установленным **Docker** и **Docker Compose** (v2).
  Проверить: `docker --version` и `docker compose version`.
- SOCKS5-прокси с выходом в регион, где открыт TikTok
  (например, прокси в РФ/СНГ, если сервер смотрит в Европу, или наоборот).

Больше ничего ставить не нужно — `python3`, `ffmpeg`, `yt-dlp`, `node` и `bun`
устанавливаются внутри образа автоматически.

---

## 1. Настройка SOCKS5-прокси

Все запросы к TikTok выполняются через `yt-dlp`, которому передаётся флаг
`--proxy=$TIKTOK_PROXY`. Без прокси TikTok, как правило, блокирует запросы.

Откройте файл `.env` в корне проекта и заполните переменные:

```dotenv
# Демо-режим: на сервере ОБЯЗАТЕЛЬНО false (иначе реальные запросы не пойдут).
DEMO_MODE=false

# SOCKS5-прокси. Формат: socks5://пользователь:пароль@хост:порт
TIKTOK_PROXY=socks5://user:pass@1.2.3.4:1080

# Сколько минут хранить скачанные видео во временном кэше (потом удаляются).
TIKTOK_CACHE_TTL_MIN=10
```

Примеры `TIKTOK_PROXY`:

| Случай                                  | Значение                                       |
|-----------------------------------------|------------------------------------------------|
| SOCKS5 с паролем                        | `socks5://user:pass@host:1080`                 |
| SOCKS5 без авторизации                  | `socks5://host:1080`                            |
| Локальный ssh-туннель (`ssh -D 1080`)   | `socks5://127.0.0.1:1080` (прокси на хосте)    |
| HTTP-прокси (yt-dlp умеет и так)        | `http://user:pass@host:8080`                    |

> Важно: если прокси живёт на хост-машине, из контейнера к нему нужно
> обращаться не по `127.0.0.1`, а по `host.docker.internal` (или по IP хоста
> в локальной сети). Пример:
> `TIKTOK_PROXY=socks5://host.docker.internal:1080`

## 2. Запуск

Из корня проекта (где лежит `docker-compose.yml`):

```bash
docker compose up -d --build
```

- `--build` — пересобирает образ (нужно при первом запуске и после обновлений).
- `-d` — в фоновом режиме.

Проверить, что контейнер поднялся:

```bash
docker compose ps
docker compose logs -f tiktok-libre
```

Открыть в браузере: **http://localhost:3000** (или `http://IP-сервера:3000`).

Остановить:

```bash
docker compose down
```

---

## 3. Где хранятся данные

Два именованных тома (см. `docker-compose.yml`):

| Том           | В контейнере       | Что хранит                                   |
|---------------|--------------------|----------------------------------------------|
| `tiktok-db`   | `/app/db`          | SQLite-база `custom.db` (подписки, видео, избранное, комментарии) |
| `tiktok-cache`| `/app/cache/videos`| Временные скачанные `.mp4` (чистятся каждые `TIKTOK_CACHE_TTL_MIN` минут) |

Посмотреть, где они лежат на диске:

```bash
docker volume inspect tiktok-libre_tiktok-db
docker volume inspect tiktok-libre_tiktok-cache
```

Удалить данные полностью (осторожно — потеряете подписки и избранное):

```bash
docker compose down -v
```

## 4. Про DEMO_MODE

`DEMO_MODE=true` — приложение отдаёт встроенные демонстрационные данные и
**не делает** реальных запросов к TikTok. Удобно, чтобы просто посмотреть на
интерфейс без прокси.

`DEMO_MODE=false` — приложение ходит в TikTok через `yt-dlp` + SOCKS5-прокси.
**На сервере должно быть именно `false`** и настроен рабочий `TIKTOK_PROXY`.

> В корневом `.env` по умолчанию стоит `DEMO_MODE=true` (это для локального
> превью в песочнице). Перед запуском на сервере **обязательно** переключите
> в `false`.

## 5. Как обновить yt-dlp

TikTok часто меняет внутренний API, поэтому `yt-dlp` нужно обновлять.
Самый простой способ — пересобрать образ (он каждый раз тянет свежий `yt-dlp`
через `pip3 install`):

```bash
docker compose up -d --build
```

Чтобы принудительно обновить без пересборки всего образа, можно зайти в
контейнер и обновить пакет, но **это изменение не переживёт пересборку**.
Правильный путь — пересобрать образ (выше) либо закрепить версию `yt-dlp`
в `Dockerfile` (`pip3 install ... yt-dlp==<version>`), если нужна стабильность.

## 6. Порт

По умолчанию приложение слушает порт **3000** внутри контейнера и маппится
на `3000` хоста (`ports: "3000:3000"` в `docker-compose.yml`).

Сменить порт на хосте (например, на 8080), не трогая образ:

```yaml
ports:
  - "8080:3000"
```

Для продакшена рекомендуется поставить перед контейнером обратный прокси
(Caddy/Nginx) с TLS. В корне проекта уже есть пример `Caddyfile`.

## 7. Переменные окружения

| Переменная               | По умолчанию              | Описание                                                             |
|--------------------------|---------------------------|----------------------------------------------------------------------|
| `DEMO_MODE`              | `false`                   | `true` — демо-данные; `false` — реальные запросы через yt-dlp.       |
| `TIKTOK_PROXY`           | _(пусто)_                 | SOCKS5-прокси для yt-dlp (`socks5://user:pass@host:port`).           |
| `TIKTOK_CACHE_TTL_MIN`   | `10`                      | Время жизни временного видео-кэша в минутах.                         |
| `TIKTOK_SOCKET_TIMEOUT`  | `30`                      | Таймаут одного запроса yt-dlp в секундах.                            |
| `TIKTOK_RETRIES`         | `3`                       | Количество ретраев yt-dlp при ошибках сети.                          |
| `DATABASE_URL`           | `file:/app/db/custom.db`  | Путь к SQLite-базе (внутри контейнера).                              |
| `TIKTOK_CACHE_DIR`       | `/app/cache/videos`       | Папка временного кэша видео (внутри контейнера).                     |
| `NODE_ENV`               | `production`              | Режим Node.js.                                                       |

Переменные можно задавать в файле `.env` (он подхватывается compose
автоматически) либо передавать через shell:

```bash
TIKTOK_PROXY=socks5://user:pass@host:1080 docker compose up -d --build
```

## 8. Частые проблемы

- **Контейнер стартует, но видео не грузятся.** Проверьте `DEMO_MODE=false`
  и что `TIKTOK_PROXY` указан и рабочий: `docker compose logs tiktok-libre |
  grep -i proxy`.
- **`bun install --frozen-lockfile` падает при сборке.** Локальный `bun.lock`
  рассинхронизирован с `package.json`. Выполните на хосте `bun install` и
  закоммитьте обновлённый `bun.lock`, затем пересоберите.
- **Прокси на хосте не доступен из контейнера.** Используйте
  `host.docker.internal` вместо `127.0.0.1` (на Linux может понадобиться
  флаг `--add-host=host.docker.internal:host-gateway` — уже поддерживается
  современными версиями Docker).
- **Хочу сбросить базу.** `docker compose down -v && docker compose up -d --build`.
